Two strings are considered **close** if you can attain one from the other using the following operations:

- Operation 1: Swap any two **existing** characters.
    - For example, `abcde -> aecdb`
- Operation 2: Transform **every** occurrence of one **existing** character into another **existing** character, and do the same with the other character.
    - For example, `aacabb -> bbcbaa` (all `a`'s turn into `b`'s, and all `b`'s turn into `a`'s)

You can use the operations on either string as many times as necessary.

Given two strings, `word1` and `word2`, return `true` _if_ `word1` _and_ `word2` _are **close**, and_ `false` _otherwise._

**Example 1:**

**Input:** word1 = "abc", word2 = "bca"
**Output:** true
**Explanation:** You can attain word2 from word1 in 2 operations.
Apply Operation 1: "abc" -> "acb"
Apply Operation 1: "acb" -> "bca"

**Example 2:**

**Input:** word1 = "a", word2 = "aa"
**Output:** false
**Explanation:** It is impossible to attain word2 from word1, or vice versa, in any number of operations.

**Example 3:**

**Input:** word1 = "cabbba", word2 = "abbccc"
**Output:** true
**Explanation:** You can attain word2 from word1 in 3 operations.
Apply Operation 1: "cabbba" -> "caabbb"
Apply Operation 2: "caabbb" -> "baaccc"
Apply Operation 2: "baaccc" -> "abbccc"

**Constraints:**

- `1 <= word1.length, word2.length <= 105`
- `word1` and `word2` contain only lowercase English letters.



# Solution 1: Hashmap

```python
from collections import defaultdict
class Solution:
    def closeStrings(self, word1: str, word2: str) -> bool:
        """
        Summary:
            Two strings are considered close if you can attain one from the other
            using the following ops:
            1. op1: swap any two existing chars
                e.g. abcde -> aecdb
            2. op2: transform every occurence of one existing character into another existing character, and do the same with the other character. 
                e.g. aacabb -> bbcbaa (all a turn into b, all b turn into a)
            
            You can use the operations on either string as many times as necessary.
        Args:
            word1 (str)
            word2 (str)
        Returns:
            true if word1 and word2 are close
            false o/w
        """
        # M0: brute force
        # edge case: 
        if len(word1) != len(word2):
            return False
        # does "close" mean == the occurence of the letters should be the same?
        # and each word have the same set of letters?
        # because you can swap infinitely times the characters
        # we reduce word to occurrences:
        """
        e.g. "a b c" {a: 1, b:1, c:1}
        e.g. "cabbba" {a: 2, b:3, c:1}
             "abbccc" {a: 1, b: 2, c:3}

        e.g. 
            "aaabbbbccddeeeeefffff" {a:3, b:4, c:2, d:2, e:5, f:5}
            "aaaaabbcccdddeeeeffff" {a:5, b:2, c:3, d:3, e:4, f:4}
            
        """
        # need to make sure 1. have the same keyset
        # 2. valueset is the same?
        # initialize values as 0.
        count1 = defaultdict(int)
        count2 = defaultdict(int)
        for i in range(len(word1)): # O(N)
            count1[word1[i]] += 1
            count2[word2[i]] += 1
        if set(count1.keys()) != set(count2.keys()):
            return False
        # if set(count1.values()) != set(count2.values()):
        #     print("set here!")
        #     return False
        # the above logic is wrong, this is not op2
        # set is wrong, need to pay attention to how to check duplicates.
        # we need to make sure for values, it is the same. 
        # pay attention to problem constraint: words only contain lowercase English letters.
        # max len of values is 26. O(26log26), which is constant. O(1). Similarly, the
        # count1 and count2 defaultdict's are also O(1)
        if sorted(count1.values()) != sorted(count2.values()): # ohh...! 
            return False

        return True
```
