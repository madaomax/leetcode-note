# [209. Minimum Size Subarray Sum](https://leetcode.com/problems/minimum-size-subarray-sum/)

Medium

## Approach: pure slide window

```python
class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        """
        Summary:
        A subarray is a *contiguous non-empty* sequence of elements within an array.

        Args:
            nums (List[int]): an arr of + int
            target (int): a + int.

        Returns:
            min len of a subarr whose sum >= target.
            0 if there is no such subarr

        Example:
            input: target = 7
            nums: [2, 3, 1, 2, 4, 3]

            input: target = 11
            nums: [1,1,1,1,1,1,1,1]
        """
        # subarray - sliding window
        # i = 0 # left ptr
        n = len(nums)
        min_len = n + 1 # return. min length of a subarr whose sum>=target
        left = 0 # left side of window
        cur_sum = 0
        if nums[0] == target:
            return 1
        for right in range(len(nums)): # O(N), right side of window
            cur_sum += nums[right]
            while cur_sum >= target: # still O(N), each elem is only iterated on once
                subarr_len = right - left + 1
                min_len = min(min_len, subarr_len)
                cur_sum -= nums[left]
                left += 1
        return min_len if min_len <= n else 0
```

When to reach for this **pattern**: 

whenever you see "minimum/maximum length subarray" with a condition that stays valid as you expand (like a sum growing), sliding window is almost always the move. The giveaway here was "contiguous subarray" + "minimum length" + all positive numbers (meaning adding elements always increases the sum, removing always decreases it — monotonic behavior that makes the window logic correct).
