# [80. Remove Duplicates from Sorted Array II](https://leetcode.com/problems/remove-duplicates-from-sorted-array-ii/)

idea: 

i as slow wrIter pointer, in this case, keeps track of the KEPT array.

j is fast reader pointer. Reads the original `nums` array.

If it is NOT a third duplicate, we write the elem into `i`. 	

```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        """
        Summary:
            Removes some duplicates in-place s.t. each unique
            elem appears at most twice.
            The relative order of the elemns should be kept the same.
            Must have the result be placed in the first part of `nums`.
            O(1) extra memory.

        Args:
            nums: List[int] arr sorted in non-decreasing order 

        Returns:
            k (int): after placing the final result in the first `k` slots
            of `nums`

        e.g.
        nums = [1,1,1,2,2,3]
        Output: 5, nums = [1,1,2,2,3,_]
        i = 2
        j = 2
        nums[j] = 
        then should proceed the reader
        """
        # Two pointers?
        # j: fast pointer reads all
        # i: selectively wrItes， slow ptr, nums[i] := where the next "kept" item should be written.
        # Because the array is sorted in non-decreasing order,
        # we should know we need to remove the 3rd same val elem, if we have it. 
        i = 2
        d = 2 # num of duplicates to keep
        # we also need to keep track of what element we are checking
        for j in range(2, len(nums)):
            if nums[j] != nums[i-d]:
                # we need to keep nums[j]
                # by replacing writer nums[i] with nums[j]
                # i keeps track of the kept sequence
                nums[i] = nums[j]
                i += 1
        return i
```

# Solution 2: Python Chad 2-liner solution

Python magic moment. 2

for each char `c` in `s`, you need `c in it` — and here's the magic: `in` on an iterator **consumes it lazily**, stopping as soon as it finds a match. So it behaves exactly like your `next` loop but in one expression.

```python
        it = iter(t)
        return all(c in it for c in s)
```
