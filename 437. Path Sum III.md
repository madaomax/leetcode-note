Given the `root` of a binary tree and an integer `targetSum`, return _the number of paths where the sum of the values along the path equals_ `targetSum`.

The path does not need to start or end at the root or a leaf, but it must go downwards (i.e., traveling only from parent nodes to child nodes).

**Example 1:**

![](https://assets.leetcode.com/uploads/2021/04/09/pathsum3-1-tree.jpg)

**Input:** root = [10,5,-3,3,2,null,11,3,-2,null,1], targetSum = 8
**Output:** 3
**Explanation:** The paths that sum to 8 are shown.

**Example 2:**

**Input:** root = [5,4,8,11,null,13,4,7,2,null,null,5,1], targetSum = 22
**Output:** 3

**Constraints:**

- The number of nodes in the tree is in the range `[0, 1000]`.
- `-109 <= Node.val <= 109`
- `-1000 <= targetSum <= 1000`

# Solution 1: DFS + prefix sum
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        """
        Summary:
            DFS? + prefix sum!
            The path does not need to start or end at the root or a leaf, but it must go downwards,
            (i.e. traveling only from parent nodes
            to child nodes)

        Args:
            `root` of a binary tree
            integer `targetSum`
        Returns:
            number of paths where the sum of the values along the path
            equals `targetSum`.
        Assumptions:
            The number of nodes in the tree is [0, 1000]
            Node.val -10^9 <= Node.val <= 10^9
            -1000 <= targetSum <= 1000
        edge case:
            0 nodes in the tree

        example:
        """

        # M0: DFS + for each path, store a prefix sum
        if root is None:
            return 0
        stack = deque() # used for DFS traversal
        stack.append((root, [root.val]))
        num_paths = 0
        while stack:
            cur, prefix_sum = stack.pop()

            # Check if there is a valid sum
            # that ends at cur node
            if prefix_sum[-1] == targetSum:
                num_paths += 1
            for i in range(len(prefix_sum) - 1): # O(depth of path, )
                if prefix_sum[-1] - prefix_sum[i] == targetSum:
                    num_paths += 1

            if cur.left is not None:
                new_prefix_sum = prefix_sum + [prefix_sum[-1] + cur.left.val]
                stack.append((cur.left, new_prefix_sum))
            if cur.right is not None:
                new_prefix_sum = prefix_sum + [prefix_sum[-1] + cur.right.val]
                stack.append((cur.right, new_prefix_sum))
        return num_paths
```



# Solution 2: DFS+prefix sum in hashtable
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        """
        Summary:
            DFS? + prefix sum!
            The path does not need to start or end at the root or a leaf, but it must go downwards,
            (i.e. traveling only from parent nodes
            to child nodes)

        Args:
            `root` of a binary tree
            integer `targetSum`
        Returns:
            number of paths where the sum of the values along the path
            equals `targetSum`.
        Assumptions:
            The number of nodes in the tree is [0, 1000]
            Node.val -10^9 <= Node.val <= 10^9
            -1000 <= targetSum <= 1000
        edge case:
            0 nodes in the tree

        example:
        """
        # M1: DFS, need to optimize prefix_sum search using a hashtable
        # therefore need DFS to be recursive

        # while we at a node, # valid paths = prefix_count[curSum - targetSum]
        
        prefix_count = defaultdict(int)
        prefix_count[0] = 1 # why do we need it?
        def dfs(node: Optional[TreeNode], cur_sum: int) -> int:
            if node is None:
                return 0
            
            cur_sum += node.val

            # check number of paths ending at this node with sum targetSum
            count = prefix_count[cur_sum - targetSum]

            # add current prefix_sum before exploring children
            prefix_count[cur_sum] += 1

            count += dfs(node.left, cur_sum)
            count += dfs(node.right, cur_sum)

            # backtrack: remove current node's prefix sum before returning
            # == this is done with cur node. 
            # !!! so smart
            prefix_count[cur_sum] -= 1

            return count
        return dfs(root, 0)

```

As we are in a BST, note that E = N - 1 where E is number of edges in a Binary Search Tree and N is number of nodes. 

DFS runtime on BST is O(V+E) = O(N + N - 1) = O(N)
At each DFS when we process node, we do O(1) hash table store and lookup

Space complexity is O(H) where H is height of BFS. Because we are storing each path's prefix sum.
