Given an `m x n` grid of characters `board` and a string `word`, return `true` _if_ `word` _exists in the grid_.

The word can be constructed from letters of sequentially adjacent cells, where adjacent cells are horizontally or vertically neighboring. The same letter cell may not be used more than once.

**Example 1:**

![](https://assets.leetcode.com/uploads/2020/11/04/word2.jpg)

**Input:** board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "ABCCED"
**Output:** true

**Example 2:**

![](https://assets.leetcode.com/uploads/2020/11/04/word-1.jpg)

**Input:** board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "SEE"
**Output:** true

**Example 3:**

![](https://assets.leetcode.com/uploads/2020/10/15/word3.jpg)

**Input:** board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]], word = "ABCB"
**Output:** false

**Constraints:**

- `m == board.length`
- `n = board[i].length`
- `1 <= m, n <= 6`
- `1 <= word.length <= 15`
- `board` and `word` consists of only lowercase and uppercase English letters.

**Follow up:** Could you use search pruning to make your solution faster with a larger `board`?



# Solution: Backtracking

```python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        rows, cols = len(board), len(board[0])

        # where i is index of the word
        def dfs(r:int, c: int, i: int) -> bool:
            # Found the whole word:
            if i == len(word):
                return True
            
            # Prune: out of bounds or char mismatch.
            if r < 0 or r>= rows or c < 0 or c >= cols or board[r][c] != word[i]:
                return False
            
            # Choose: mark current cell as visited
            tmp = board[r][c]
            board[r][c] = '#'

            # Explore: 4 directions
            found = (dfs(r+1, c, i+1) or
                     dfs(r-1, c, i+1) or
                     dfs(r, c+1, i+1) or
                     dfs(r, c-1, i+1))

            # un-choose/un-mark, restore cell
            board[r][c] = tmp
            return found

        # multi-source DFS
        for r in range(rows):
            for c in range(cols):
                if dfs(r, c, 0):
                    return True

        return False
```

**Why it's backtracking.** From any starting cell you try to match the word character by character, branching into the 4 neighbors. When a path dead-ends (mismatch, out of bounds, or revisiting a cell), you undo your last step and try a different direction. That "mark, explore, unmark" is the backtracking signature.

**Approach.** Try every cell as a possible start. From each, run a DFS that matches `word[index]` against the current cell, then recurses into neighbors for `index + 1`. To enforce the no-reuse rule, mark the current cell as visited before recursing and restore it after.

The key trick is marking visited _in place_: temporarily overwrite the cell with a sentinel character (like `#`) so it can't be matched again in the current path, then put the original character back when you backtrack. This avoids a separate visited set.

```python
def exist(board, word):
    rows, cols = len(board), len(board[0])

    def dfs(r, c, i):
        # Found the whole word
        if i == len(word):
            return True
        # Prune: out of bounds or character mismatch
        if r < 0 or r >= rows or c < 0 or c >= cols or board[r][c] != word[i]:
            return False

        # Choose: mark current cell visited
        tmp = board[r][c]
        board[r][c] = '#'

        # Explore: 4 directions
        found = (dfs(r + 1, c, i + 1) or
                 dfs(r - 1, c, i + 1) or
                 dfs(r, c + 1, i + 1) or
                 dfs(r, c - 1, i + 1))

        # Un-choose: restore cell
        board[r][c] = tmp
        return found

    for r in range(rows):
        for c in range(cols):
            if dfs(r, c, 0):
                return True
    return False
```

**Mapping to the template.** `i == len(word)` is `is_solution`. The bounds/mismatch check is the pruning (`is_valid`). Overwriting with `#` is _choose_, the four recursive calls are _explore_, and restoring `tmp` is _un-choose_.

**Trace** on `board = [["A","B"],["C","D"]]`, `word = "ABD"`:

- Start at (0,0)`A`, matches `word[0]`. Mark it `#`, recurse for index 1.
- Try down → (1,0)`C` ≠ `B`, fail. Try up → out of bounds. Try right → (0,1)`B` = `word[1]`. Mark it, recurse for index 2.
    - From (0,1): down → (1,1)`D` = `word[2]`, recurse for index 3 → `i == len(word)`, return `True`. Chain unwinds as `True`.

**Complexity.** Time is O(m · n · 4^L) where L is the word length: m·n starting cells, and each DFS branches up to 4 ways (really 3 after the first step, since you don't immediately revisit) to depth L. Space is O(L) for the recursion stack, since marking in place uses no extra grid.

**Easy pruning win:** if `len(word) > rows * cols`, return `False` immediately. A stronger one: count character frequencies in the board and bail if any letter in `word` appears more often than the board has, and optionally start the search from whichever end of `word` is rarer in the grid.
