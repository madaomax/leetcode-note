# [339. Nested List Weight Sum](https://leetcode.com/problems/nested-list-weight-sum/)



## Sol: BFS / DFS

Intuition -> something nested -> DFS

depth -> BFS

so both are fine!

``````python
class Solution:
    def depthSum(self, nestedList: List[NestedInteger]) -> int:
        """
        Summary:
            depth of an integer is the number of lists that it is inside of. 
        Args:
            nestedList: a list of integers. Each elemnt is either an integer or a list
                        whose elements may also be integers or other lists.
        
        Returns:
            the sum of each integer in nestedList multiplied by its depth
        """
        result = 0
        # BFS template
        # list-like container with fast appends and pops on either end
        queue = deque(nestedList)
        depth = 1
        while queue:
            for _ in range(len(queue)):
                node = queue.popleft()
                # Logic
                if node.isInteger():
                    result += depth * node.getInteger()
                else:
                    queue.extend(node.getList())
            depth += 1
        return result


    # quickly! dfs implementation
        def dfs(nested_list, depth):
            total = 0
            for nested in nested_list:
                if nested.isInteger():
                    total += nested.getInteger() * depth
                else:
                    total += dfs(nested.getList(), depth + 1)
            return total
        return dfs(nestedList, 1)

``````

Python trick: use [deque](https://docs.python.org/3/library/collections.html#collections.deque) instead of a list! Deque has fast appends and pops on either side.
