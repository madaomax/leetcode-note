Given an array of meeting time intervals `intervals` where `intervals[i] = [starti, endi]`, return _the minimum number of conference rooms required_.

**Example 1:**

**Input:** intervals = [[0,30],[5,10],[15,20]]
**Output:** 2

**Example 2:**

**Input:** intervals = [[7,10],[2,4]]
**Output:** 1

**Constraints:**

- `1 <= intervals.length <= 104`
- `0 <= starti < endi <= 106`

# Solution 1: one-pass, min-heap
```python
class Solution:
    def minMeetingRooms(self, intervals: List[List[int]]) -> int:
        """
        Summary:

        Args:
            an array of meetings intervals
        Returns:
            The minimum number of conference room needed.
        Assumptions:
        1 <= intervals.length <= 10^4
        0 <= start_i < end_i <= 10^6

        This feels like counter part of merge intervals.
        Should we maintain heaps and categorize each interval?
        """
        intervals.sort(key=lambda x: x[0])
        # if two intervals have overlapping time, then add the interval to another queue?
        # but we only need to return NUMBER of conference rooms tho
        # like, 0, 1, 2, 3?
        # idk
        # num_rooms = 0
        # for interval in intervals:
        #     if not rooms:
        #         # rooms.append(interval)#
        #         rooms.append(interval[1])
        #         num_rooms += 1
        #         continue
        #     # need to keep track of each rooms'
        #     # end time
        #     # can only append a new room when the
        #     # end time is no later than the start time
        #     for room in rooms:
        #         # return the smallest meeting room
        #         # end time?
        heap = []
        for start, end in intervals: # O(N)
            if heap and heap[0] <= start:
                heapq.heappop(heap) # O(logn)
            heapq.heappush(heap, end) # O(logN)
        return len(heap)
```

The idea is to we need to keep track of the end time for each room, and when we want to get a room, we want to search for "is there a room that is already freed?". Therefore, we can search for a room that has minimum end time. (greedy?)

To search for a room that has minimum end time across some rooms, which is, finding a minimum value among k values. This brings us to min-heap.  [[0. Heap]]
which saves O(N) of search time to O(logN).

Runtime: O(NlogN)
Space Complexity: O(N) for maintaining the heap.
