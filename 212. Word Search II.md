Given an `m x n` `board` of characters and a list of strings `words`, return _all words on the board_.

Each word must be constructed from letters of sequentially adjacent cells, where **adjacent cells** are horizontally or vertically neighboring. The same letter cell may not be used more than once in a word.

**Example 1:**

![](https://assets.leetcode.com/uploads/2020/11/07/search1.jpg)

**Input:** board = [["o","a","a","n"],["e","t","a","e"],["i","h","k","r"],["i","f","l","v"]], words = ["oath","pea","eat","rain"]
**Output:** ["eat","oath"]

**Example 2:**

![](https://assets.leetcode.com/uploads/2020/11/07/search2.jpg)

**Input:** board = [["a","b"],["c","d"]], words = ["abcb"]
**Output:** []

**Constraints:**

- `m == board.length`
- `n == board[i].length`
- `1 <= m, n <= 12`
- `board[i][j]` is a lowercase English letter.
- `1 <= words.length <= 3 * 104`
- `1 <= words[i].length <= 10`
- `words[i]` consists of lowercase English letters.
- All the strings of `words` are unique.

# Solution: Trie + DFS/backtracking

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.word = None # Holds the full word if a word ends here

class Solution:
    def findWords(self, board: List[List[str]], words: List[str]) -> List[str]:
        # Build Trie based on words
        root = TrieNode()
        for word in words:
            node = root
            for ch in word:
                if ch not in node.children:
                    node.children[ch] = TrieNode()
                node = node.children[ch]
            node.word = word

        rows, cols = len(board), len(board[0])

        result = []

        def dfs(r, c, node):
            ch = board[r][c]
            # prune: cur char not a viable next step in any word
            if ch not in node.children:
                return
            nxt = node.children[ch]

            # found a complete word
            if nxt.word is not None:
                result.append(nxt.word)
                nxt.word = None # dedup: don't add the same word twice

            # choose: mark visited # oh yeah visited is ok, it's still valid
            # in the trie for its children
            board[r][c] = '#'

            for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols and board[nr][nc] != '#':
                    dfs(nr, nc, nxt)

            # Un-choose: restore
            board[r][c] = ch

            # Optimal optimization: prune dead leaves
            # why? is it because the word has found already? 
            if not nxt.children:
                node.children.pop(ch)
            
        for r in range(rows):
            for c in range(cols):
                dfs(r,c,root)
        return result
```


**The naive idea and why it's bad.** You could just run the LC 79 solution once per word. That's O(W · m · n · 4^L) where W is the number of words. When you have thousands of words, you're re-walking the same grid paths over and over, wastefully.

**The key insight.** Instead of searching for one word at a time, search for _all words simultaneously_ in a single DFS. To do that, you need a structure that lets you check "is the path I've built so far a prefix of any word?" cheaply. That's exactly what a **trie** gives you.

So the plan: build a trie from the word list, then DFS the grid where at each cell you walk _down the trie in lockstep_ with the path. If the current character isn't a child of the current trie node, the whole subtree of grid paths is dead, prune immediately. This is the big win: one mismatch prunes against all words at once.

**Building the trie.** Each node holds a dict of children. The neat trick is storing the complete word at the node where it ends (instead of a boolean `isWord` flag), so when you reach it you can directly collect it.

**What changed versus LC 79.** The recursion now carries a _trie node_ instead of a word index. The match check became "is this character a child of the current node" instead of "does it equal `word[i]`". And instead of returning a bool up the chain, you collect into `result` whenever you land on a node whose `word` is set.

**Three details that matter.**

The de-dup line `nxt.word = None` after appending. The same word can be reachable by multiple grid paths, and you only want it once. Setting `word` to `None` after collecting means subsequent arrivals at that node won't re-add it. Cleaner than maintaining a separate set.

The leaf-pruning at the bottom, `if not nxt.children: node.children.pop(ch)`. Once a trie node has no children left (all its words have been found and it's not a prefix of anything remaining), you snip it from the tree. As the search progresses the trie shrinks, so later DFS calls prune even faster. This is the optimization that separates a passing solution from a fast one on LeetCode.

Marking visited in place with `#`, same as before, and I fold the bounds check into the neighbor loop so the recursion always starts on a valid, unvisited cell.

**Trace** on `board = [["o","a"],["e","t"]]`, `words = ["oat","eat"]`:

- Trie: root → `o`→`a`→`t`(word="oat"), root → `e`→`a`→`t`(word="eat").
- Start (0,0)`o`: it's a child of root. Mark `#`. Neighbors: down (1,0)`e` not a child of node `o`, prune. Right (0,1)`a` is a child of `o`. Recurse.
    - (0,1)`a`: child of `o`, advance to node `a`. Mark `#`. Neighbor down (1,1)`t` is a child of `a`, and that node has `word="oat"`. Collect "oat", set its word to `None`. Restore.
- Eventually start (1,0)`e`: child of root → node `e`. Right (1,1)`t` not a child of `e`, prune. Up (0,0)`o` not a child of `e`, prune. Hmm, "eat" needs `e→a→t` but `a` isn't adjacent to this `e` here, so it's correctly not found in this layout.

**Complexity.** Building the trie is O(total characters across all words). The DFS is O(m · n · 4^(L−1)) in the worst case where L is the longest word length, but in practice the trie pruning makes it dramatically faster because most grid paths die after a few characters. Space is O(total characters) for the trie plus O(L) recursion depth.

**The interview talking point.** When you recognize "find many words/patterns in one structure," the move is almost always _trie + DFS with shared prefixes_. The single most important sentence to say out loud: "rather than search per-word, I build a trie so one grid traversal prunes against all words simultaneously, and a character mismatch kills the entire subtree of paths." That framing alone signals you've seen the pattern.
