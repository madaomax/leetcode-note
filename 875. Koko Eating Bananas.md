Koko loves to eat bananas. There are `n` piles of bananas, the `ith` pile has `piles[i]` bananas. The guards have gone and will come back in `h` hours.

Koko can decide her bananas-per-hour eating speed of `k`. Each hour, she chooses some pile of bananas and eats `k` bananas from that pile. If the pile has less than `k` bananas, she eats all of them instead and will not eat any more bananas during this hour.

Koko likes to eat slowly but still wants to finish eating all the bananas before the guards return.

Return _the minimum integer_ `k` _such that she can eat all the bananas within_ `h` _hours_.

**Example 1:**

**Input:** piles = [3,6,7,11], h = 8
**Output:** 4

**Example 2:**

**Input:** piles = [30,11,23,4,20], h = 5
**Output:** 30

**Example 3:**

**Input:** piles = [30,11,23,4,20], h = 6
**Output:** 23

**Constraints:**

- `1 <= piles.length <= 104`
- `piles.length <= h <= 109`
- `1 <= piles[i] <= 109`

# Solution 1: Brute Force
```python
import math
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        """
        Summary:
        Each hour, she chooses some pile of bananas and eats `k` bananas from that pile.
        If the pile has < k bananas, she eats all of them instead and will not eat any more bananas during this hour.

        Args:
            piles: n piles of bananas. ith pile has piles[i] bananas.
            h: in h hours the guards will be back. 
        Returns
            min integer `k` s.t. she can eat all the bananas within h hours.
        """
        # M0: brute force
        # When actually doing the problem, start with brute force. 
        speed = 1
        while True:
            hour_spent = 0
            for pile in piles:
                hour_spent += math.ceil(pile / speed)
            
           # if koko can finish all the piles within h hours, if so, return speed.
            # o/w, let speed increment by 1 and repeat the previous iteration. 
            if hour_spent <= h:
                return speed
            else:
                speed += 1
        # time complexity: final speed = m, n = len(piles). O(mn)
        # space complexity: O(1)
```

After we implement this approach, we can find that this is testing a range of possible values, from 1 all the way to max(piles), finding the minimum value that satisfies a certain criteria in this range. 
The problem also shows us two properties: 1. for a speed k, if k works, then k + 1 also works. 2. for a speed k, if k does not work, then k - 1 also does not work. 
Therefore, the array is formatted as the values on the left side of k are invalid values and valid values on its right side, which looked like binary search. Binary Search is always quite good because it turns a O(N) operation to O(logN) with no tradeoff. 

# Solution 2: Binary Search
```python
import math
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        """
        Summary:
        Each hour, she chooses some pile of bananas and eats `k` bananas from that pile.
        If the pile has < k bananas, she eats all of them instead and will not eat any more bananas during this hour.

        Args:
            piles: n piles of bananas. ith pile has piles[i] bananas.
            h: in h hours the guards will be back. 
        Returns
            min integer `k` s.t. she can eat all the bananas within h hours.
        """
        # M1: Binary Search
        # Initialize boundaries
        left = 1
        right = max(piles)

        while left < right: # O(logm) where m = max(piles)
            middle = (left + right) // 2
            hour_spent = 0

            for pile in piles: # O(n)
                hour_spent += math.ceil(pile / middle)

            # check if middle is a workable speed, and cut the search space by half. 
            if hour_spent <= h:
                right = middle
            else:
                left = middle + 1

        # only when the left and right boundaries coincide, we find the target value,
        # that is, the minimum workable eating speed.
        return right

        # Time complexity:O(nlogm). Let max(piles) = m. 
        # It takes logm comparisons to reduce the search space to 1.
        # for each eating speed `middle`, we traverse the array and calculate the overall time
        # koko spends, which takes O(n) for each traversal

        # space complexity O(1)
```
