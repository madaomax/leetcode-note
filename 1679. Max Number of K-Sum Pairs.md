https://leetcode.com/problems/max-number-of-k-sum-pairs/description/
Medium
You are given an integer array `nums` and an integer `k`.

In one operation, you can pick two numbers from the array whose sum equals `k` and remove them from the array.

Return _the maximum number of operations you can perform on the array_.

**Example 1:**

**Input:** nums = [1,2,3,4], k = 5
**Output:** 2
**Explanation:** Starting with nums = [1,2,3,4]:
- Remove numbers 1 and 4, then nums = [2,3]
- Remove numbers 2 and 3, then nums = []
There are no more pairs that sum up to 5, hence a total of 2 operations.

**Example 2:**

**Input:** nums = [3,1,3,4,3], k = 6
**Output:** 1
**Explanation:** Starting with nums = [3,1,3,4,3]:
- Remove the first two 3's, then nums = [1,4,3]
There are no more pairs that sum up to 6, hence a total of 1 operation.

**Constraints:**

- `1 <= nums.length <= 105`
- `1 <= nums[i] <= 109`
- `1 <= k <= 109`


# Solution
## approach 1: hashmap
```python
from collections import defaultdict

class Solution:
    def maxOperations(self, nums: List[int], k: int) -> int:
        """
        Summary:
            In one op, you can pick two numbers from the array whose sum equals `k` and remove
            them from the array.
        Args:
            int array `nums`
            integer `k`
        Return:
            maximum number of operations you can perform on the array
        E.g.:
            nums = [1,2,3,4]
            k = 5
                - 1, 4 => [2,3]
                - 2, 3 => []
            Returns: 2
        
        e.g.:
            nums = [3,1,3,4,3]
              - first 2 3s [1, 4, 3]
            Returns:
                3

        e.g. 
            [1 ,4,4,1 ,2 ,1,2,2,3,2,4,2]
            ret = 1
        """

        # returns max#op, no need to record each op.
        # M0: do a 1-pass of the array and try to find target?
        num_ops = 0
        can_be_first_of_op = defaultdict(int) # enable O(1) search
        # initialize each value to be int 0
        for num in nums: # O(N)
            if num < k: # because 1 <= nums[i]
                # if it's first number is already there:
                if can_be_first_of_op.get(k-num, 0) > 0:
                    num_ops += 1
                    # remove entry as this pair works already
                    can_be_first_of_op[k - num] -= 1 # O(1)
                else:
                    # want to find target this first num
                    # count occurence of num. In case of duplicate entries.
                    can_be_first_of_op[num] += 1 # O(1)
        return num_ops

```

Hashmap single pass.
Time complexity O(N)
Space complexity O(N)

A slightly worse runtime method O(nlogn) using two pointers:
1. sort first (O(nlogn))
2. have a pointer at the left, and one at the right. O(N)
