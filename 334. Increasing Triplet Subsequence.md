
[334. Increasing Triplet Subsequence](https://leetcode.com/problems/increasing-triplet-subsequence/)

Medium

Given an integer array `nums`, return `true` _if there exists a triple of indices_ `(i, j, k)` _such that_ `i < j < k` _and_ `nums[i] < nums[j] < nums[k]`. If no such indices exists, return `false`.

**Example 1:**

**Input:** nums = [1,2,3,4,5]
**Output:** true
**Explanation:** Any triplet where i < j < k is valid.

**Example 2:**

**Input:** nums = [5,4,3,2,1]
**Output:** false
**Explanation:** No triplet exists.

**Example 3:**

**Input:** nums = [2,1,5,0,4,6]
**Output:** true
**Explanation:** One of the valid triplet is (1, 4, 5), because nums[1] == 1 < nums[4] == 4 < nums[5] == 6.

**Constraints:**

- `1 <= nums.length <= 5 * 105`
- `-231 <= nums[i] <= 231 - 1`

**Follow up:** Could you implement a solution that runs in `O(n)` time complexity and `O(1)` space complexity?


## Solution


```python
class Solution:
    def increasingTriplet(self, nums: List[int]) -> bool:
        """
        Summary"

        args:
            `nums`: int array

        Returns:
            True  if there exists a triple of indices (i,j,k)
                  s.t. i < j < k and nums[i] < nums[j] < nums[k]
            False if there is no such indices
        """
        # M1: brute force? (O(N^3))
        # this gets time limit exceeded
        # aka. if there are no triplets that are monotonally increasing
        # then return falase
        # for i in range(len(nums)):
        #     for j in range(i+1, len(nums)):
        #         for k in range(j+1,len(nums)):
        #             if nums[i] < nums[j] < nums[k]:
        #                 return True
        # return False

        # M2: quicker. Ad-hoc. See what info can we extract by only one-pass
        # mentally do a one pass to figure out the solution. 
        first = float('inf') # min number met
        second = float('inf')  # min number > first, whose index > first's

        for cur in nums: # O(N)
            if cur <= first:
                # will always update the the first number
                first = cur
            elif cur <= second:
                # if first < cur <= second
                second = cur
            else:
                # if first < cur < second
                return True
        return False
```


Type of problem: Array

How to solve:
Find variables to keep track of while looping through the array. 
Kind of an ad-hoc exploration. 

Note that we don't need actual numbers, only need to return True or False.  => Takeaway: core pattern is "maintaining a running optimal state". 
No need to store data, only need to scan left to right and use a few variables to remember "the most favorable situation seen so far". That "most favorable" value is often a *minimum*, because the smaller a value you keep around, the easier it is for something later to build on top of it. 

We only need to make sure, there is a valid **second** number, such that there is a number to its left that is smaller, and a number to its right that is larger. 

Also, we want to make **first** and **second** as small as possible so it is easier to find the third number that can form a valid sequence. 

Additionally, when using variables, think about how to initialize them. Use `float('inf') or float(-'inf')` to skip "is there a valid?" check. Other initialization  values: 0, 1, etc.

`<=` vs `<` for handling duplicate elements.
Build a habit of quickly constructing an "all equal" or "has duplicates" test case and running it through. This is a hotspot for edge-case bugs on LC.
